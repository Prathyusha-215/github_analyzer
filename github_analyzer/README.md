# GitHub Repository Analyzer

A full-stack, enterprise-grade AI-powered codebase analyzer that evaluates GitHub repositories dynamically. It provides a real-time web interface, dynamic repository fetching, intelligent feedback generated by GROQ LLMs, and comprehensive metrics.

## Features
- **Real-Time Analysis**: Supports analyzing GitHub repos directly via the web interface.
- **Batch Evaluation**: Allows uploading Excel sheets to grade bulk repositories (like hackathons or classes).
- **Infinite Depth**: Uses Git Trees payload avoiding rate limits and recursion issues.
- **Language Coverage**: Recognizes 40+ languages seamlessly.
- **Smart Feedback**: Detailed UI showing repo metrics, file explorers, and AI-driven code reviews.

## Architecture
- **Backend:** FastAPI, Python, PyGithub, Groq SDK.
- **Frontend:** Jinja2 Templates, HTML, Tailwind CSS.

## Local Setup

1. **Clone the repository:**
   ```bash
   git clone <your-repo-url>
   cd github_analyzer
   ```

2. **Install Dependencies:**
   ```bash
   pip install -r requirements.txt
   ```

3. **Environment Setup:**
   Create a `.env` file in the root directory and add the following keys:
   ```env
   GITHUB_TOKEN=your_github_personal_access_token
   GROQ_API_KEY=your_groq_api_key
   ```

4. **Run Locally:**
   ```bash
   uvicorn main:app --host 0.0.0.0 --port 8000 --reload
   ```

5. **Visit Open the App:** Go to `http://localhost:8000`.

## Deployment to Render

This project is configured out-of-the-box to deploy smoothly on [Render](https://render.com).

### Option 1: Render Blueprints (Automatic)
We have included a `render.yaml` file for unified 1-click infrastructure deployment.
1. Create an account on Render.
2. Go to the "Blueprints" tab.
3. Click "New Blueprint Instance" and link your GitHub repository.
4. Render will automatically detect the settings. **Make sure to manually enter the values for `GITHUB_TOKEN` and `GROQ_API_KEY` when prompted in the Render Dashboard** (as they are intentionally excluded from the blueprint sync for security).

### Option 2: Manual Web Service
1. On the Render Dashboard, click **New +** and select **Web Service**.
2. Connect your GitHub repository.
3. Use the following configuration:
   - **Environment:** `Python`
   - **Build Command:** `pip install -r requirements.txt`
   - **Start Command:** `uvicorn main:app --host 0.0.0.0 --port $PORT`
4. Expand the **Advanced** section and add your Environment Variables (`GITHUB_TOKEN`, `GROQ_API_KEY`).
5. Click **Create Web Service**.

> **Note:** Render handles background tasks natively on persistent/ephemeral SSDs without freezing, completely resolving previous deployment restrictions on Vercel.
